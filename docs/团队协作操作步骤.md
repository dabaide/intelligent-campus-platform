---

## 前置：仓库所有者（tao0012）需要做的事

### 1. 添加协作者

打开 https://github.com/tao0012/intelligent-campus-platform/settings/access

- 点击 **Add people**
- 分别输入 B、C、D 的 GitHub 用户名
- 角色选 **Write**（允许他们直接推送）
- 每人会收到 GitHub 邮件邀请，**必须点击接受**

### 2. 把这个文档发给每个人

## 所有人（A/B/C/D）统一操作步骤

### 第一步：安装 Git

从 https://git-scm.com/download/win 下载安装，一路默认即可。安装完成后打开 **Git Bash**，以下所有命令在 Git Bash 中执行。

### 第二步：配置身份（第一层证据）

```bash
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"
```

验证：
```bash
git config --global user.name
git config --global user.email
```

### 第三步：生成 SSH 密钥（第二层证据）

```bash
ssh-keygen -t ed25519 -C "你的GitHub注册邮箱"
# 一路回车即可，默认保存在 ~/.ssh/id_ed25519
```

查看并复制公钥：
```bash
cat ~/.ssh/id_ed25519.pub
```

把输出的内容**整行复制**，然后：
1. 打开 https://github.com/settings/keys
2. 点击 **New SSH Key**
3. Title 填 `自己的电脑`
4. Key 粘贴刚复制的内容
5. 点击 **Add SSH Key**

测试连接：
```bash
ssh -T git@github.com
```
看到 `Hi 你的用户名! You've successfully authenticated` 即成功。

### 第四步：克隆仓库

```bash
cd ~/Desktop
git clone git@github.com:tao0012/intelligent-campus-platform.git
cd intelligent-campus-platform
```

### 第五步：按分工修改自己的文件

各人只修改自己负责的文件（详见下方分工）。

### 第六步：提交并推送

```bash
git add .
git commit -m "A：完成导航智能体和学术咨询智能体"
git push origin main
```

> ⚠️ commit 信息要写清楚谁、做了什么，方便答辩时追溯。

## 日常协作流程

每次开始工作前：

```bash
cd intelligent-campus-platform
git pull origin main        # 拉取最新代码
```

每次完成一个功能后：

```bash
git add .
git commit -m "B：完成行政事务智能体-XXX功能"
git push origin main
```

如果 push 时提示冲突：

```bash
git pull origin main        # 先拉取合并
# 解决冲突后
git add .
git commit -m "合并冲突解决"
git push origin main
```

## 答辩时如何证明贡献

1. GitHub 仓库 → **Insights → Contributors**，每人头像和提交次数一目了然
2. 每次 commit 绑定了各人的 GitHub 账号（第一层）
3. 每次 push 带有各人的 SSH 指纹（第二层）
4. 具体改了哪些文件：GitHub 上点每个 commit 即可查看增减内容
