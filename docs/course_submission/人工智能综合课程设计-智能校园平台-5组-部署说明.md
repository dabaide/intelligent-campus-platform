## 三、系统部署概述

### 3.1 系统形态说明
本项目属于以下类别：
- [x] Web 前后端系统
- [x] 多智能体协同系统
- [x] 大模型/RAG 应用系统

### 3.2 部署方式说明
本项目采用的部署方式：
- [x] 本地部署
- [x] Docker 容器部署

### 3.3 部署架构说明
本系统的部署架构如下：
- **前端部署位置**：本地 Node.js 环境或 Nginx 容器，运行 React 前端页面，默认端口 3000。
- **后端服务部署位置**：本地 Python 环境或 FastAPI 容器，默认端口 8000。
- **模型服务部署位置**：通过公网调用第三方大模型 API (DeepSeek)。
- **数据库/缓存/向量库部署位置**：本地或容器内运行的 Redis（缓存与会话状态管理）；本地文件系统中的 FAISS（向量数据库，存储于 `backend/data/vector_store`）。
- **第三方接口依赖**：依赖外部 LLM API 服务，需联网访问。
- **硬件设备接入方式**：无特定硬件依赖。

## 六、标准部署流程

### 6.1 获取项目代码
```bash
git clone https://gitee.com/your-repo/intelligent-campus-platform.git
cd intelligent-campus-platform
```
*(如为离线提交版本，请解压缩代码包并进入解压后的 intelligent-campus-platform 目录。)*

### 6.2 创建运行环境
**后端环境配置：**
```bash
cd backend
python -m venv venv
# Windows:
.\venv\Scripts\activate
# Linux/Mac:
# source venv/bin/activate

pip install -r requirements.txt
```

**前端环境配置：**
```bash
cd ../frontend
npm install
```

### 6.3 配置环境变量
在 `backend` 目录下复制 `.env.example` 并重命名为 `.env`。
配置示例：
```env
# 基本配置
PROJECT_NAME=Intelligent Campus Platform
DEBUG=true
LOG_LEVEL=INFO

# 数据库配置
DATABASE_URL=sqlite+aiosqlite:///./intelligent_campus.db
REDIS_URL=redis://localhost:6379/0

# AI服务配置
DEEPSEEK_API_KEY=sk-xxxxxx-your-deepseek-api-key-xxxxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
```

### 6.4 初始化数据库或知识库
确保本地配置无误后，初始化本地向量检索库及演示账号。如果您曾在之前运行过测试，建议先清理旧的缓存：
```bash
# Windows 下可执行根目录提供的清理脚本
.\clear_all_cache.ps1

# 然后初始化数据库及文档
cd backend
python scripts/init_demo_users.py
python scripts/import_hbue_knowledge.py
```

### 6.5 下载或加载模型文件
- **模型文件来源**：本系统使用 DeepSeek API 调用，不需要下载百 GB 级的大模型权重。
- **文件名称与目录位置**：本地向量库为基于 CPU 的 FAISS 索引文件，会在执行 6.4 步时自动在 `backend/data/vector_store` 生成。
- **首次加载时间**：无首批大模型加载时间问题，向量库毫秒级加载。

### 6.6 启动服务
请按推荐顺序开启多个终端窗口启动：

**1. 启动 Redis 服务**
- Windows 推荐使用 memurai 或 WSL 启动 `redis-server`。
- Linux/Mac 运行 `redis-server`。

**2. 启动后端**
```bash
cd backend
# 确保处于 venv 激活状态
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

**3. 启动前端**
```bash
cd frontend
npm run dev
```

*(如使用 Docker 一键部署：在项目根目录运行 `docker-compose up -d --build`)*

## 七、不同项目类型的部署补充说明

### 7.1 Web 系统类项目
- **架构**：采用前后端分离部署。
- **接口跨域配置**：FastAPI 配置了 CORSMiddleware，默认允许前端 http://localhost:3000 跨域请求。
- **前端构建命令**：生产环境执行 `npm run build`。
- **Nginx 或静态资源部署方式**：构建产出 `build` 文件夹可直接放入 Nginx 的 `html` 目录下代理。
- **默认访问地址**：`http://localhost:3000`。

### 7.2 多智能体 / RAG 类项目
- **所依赖的大模型调用方式**：通过 LangChain/CrewAI 发起 REST HTTP 请求对接 DeepSeek API。
- **向量库初始化方法**：利用 `FAISS.from_documents()` 本地构建。
- **Prompt 配置位置**：后端 `backend/agents/prompts.py` 中。
- **API Key 配置方法**：环境变量 `.env`。
- **离线与在线模式切换方法**：若断网，可在代码中开启 Mock 模式返回硬编码结果。

### 7.3 视觉检测 / 视频分析类项目
*(本项目不涉及计算机视觉与视频流处理，此项不适用。)*

### 7.4 边缘部署 / Jetson / 嵌入式类项目
*(本项目不涉及边缘硬件设备，此项不适用。)*

### 7.5 无人机 / 传感器联动类项目

## 八、验收现场部署建议

### 8.1 现场部署策略
- 优先使用已验证成功的稳定版本；
- 尽量减少临时联网下载依赖，推荐直接使用打包好 `venv` 和 `node_modules` 的离线包；
- 提前设置好默认账号与演示环境；
- 检查验收现场网络是否支持访问 DeepSeek API 域名，避免网络拦截。

### 8.2 最小可演示部署方案
- 至少保证本地前后端通信正常，能够演示智能体流转过程。
- 确保至少加载了一个样例知识库文档（如学生手册），以演示 RAG 的功能。

### 8.3 应急预案
- **主机故障时**：切换到预装了相同环境的备用电脑；
- **外网异常或 API 不可用时**：后端切换为内置缓存数据演示模式，或者使用提前录制好的演示视频。
- **Redis 故障时**：清除历史会话功能，仅演示单轮问答。

## 十、常见部署问题与解决方案
| 问题现象 | 可能原因 | 解决方法 |
| --- | --- | --- |
| 依赖安装失败 | Python/Node 版本不匹配 | 使用指定版本并重新安装依赖 |
| 模型无法加载 | 文件路径错误或权重缺失 | 检查知识库向量路径与配置文件 |
| 页面打不开 | 前端未启动或端口冲突 | 重启服务并更换端口 3000 |
| 接口报错 500 | SQLite 并发锁定或配置错误 | 若频繁出现 `database is locked`，请确认 `aiosqlite` 是否已安装，或者执行 `clear_all_cache.ps1` 后重启。 |
| 数据库连接失败 | 地址、账号或权限错误 | 核对数据库参数或 SQLite 写入权限 |
| 知识检索报错 | FAISS 包缺失 | Windows 环境建议 `pip install faiss-cpu` |
| API 调用失败 | Key 失效或网络异常 | 检查 DeepSeek 余额或系统代理，更换 Key |

## 十二、回滚与卸载说明

### 12.1 服务停止方法
- **后端停止**：在运行的终端按 `Ctrl+C`。如需强杀：`pkill -f python` (Linux/Mac) 或在任务管理器结束 Python 进程。
- **前端停止**：在终端按 `Ctrl+C`。如需强杀：`pkill -f node`。
- **Redis 停止**：关闭 redis-server 的终端。

### 12.2 数据回滚方法
- **需要重置向量知识库**：直接删除 `backend/data/vector_store` 文件夹，然后重新执行初始化脚本即可。

### 12.3 环境清理方法
```bash
# 清理后端虚拟环境
rm -rf backend/venv

# 清理前端依赖
rm -rf frontend/node_modules

# 容器部署清理
docker-compose down -v
```
